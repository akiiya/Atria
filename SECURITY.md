## 报告安全漏洞

如果您发现 Atria 的安全漏洞，请私下报告给维护者。

**请不要公开提交包含以下内容的 issue：**

- 真实 API Hash
- 真实 Session 数据
- 真实手机号
- 真实验证码
- 真实 2FA 密码
- secret.key 内容

### 报告方式

发送邮件至：[security@example.com](mailto:security@example.com)

（注：发布前需要替换为真实安全联系邮箱）

### 报告内容

请包含：
- 漏洞描述
- 复现步骤
- 潜在影响
- 建议修复方案（如有）

### 响应时间

- 确认收到：48 小时内
- 初步评估：7 天内
- 修复发布：根据严重程度，14-30 天内

## 安全设计说明

### 数据保护

- **密码**：使用 bcrypt 哈希存储，永不保存明文
- **API Hash**：AES-256-GCM 加密存储，永不记录到日志
- **手机号**：AES-256-GCM 加密存储，日志只记录脱敏版本
- **Session 文件**：AES-256-GCM 加密存储
- **验证码/2FA 密码**：不保存、不记录、不写入审计日志

### 密钥管理

- `secret.key` 是所有加密数据的唯一解密密钥
- 丢失 `secret.key` 将导致加密数据无法恢复
- 请务必备份 `secret.key` 到安全位置

### 数据目录

`data/` 目录包含敏感数据，**不得提交到代码仓库**：
- `atria.db` — SQLite 数据库
- `secret.key` — 加密密钥
- `sessions/` — 加密的 Session 文件
- `logs/` — 应用日志

### 非目标

Atria **不是**以下用途的工具：

- 垃圾消息或未经请求的消息
- 批量邀请成员或加入群组
- 绕过平台限制
- 账号交易或账号市场
- 手机号采集或接码平台
- 任何形式的自动化骚扰
- 刷粉、刷浏览、刷互动

## 安全更新

- 安全补丁优先于功能开发
- 通过 GitHub releases 通知安全更新
- 关键漏洞立即发布补丁版本
