# Atria — UI 设计指南

## 1. 设计目标

Atria 的 Web 界面是一个现代化的全屏管理面板，用于管理 MTProto Session、API 凭据、账号信息和系统配置。

设计原则：
- **大方** — 充足的留白和清晰的信息层级
- **简洁** — 不追求花哨动画，注重实用性
- **现代** — 符合当代 Web 应用的视觉标准
- **实用** — 每个元素都有明确的功能目的

## 2. 全屏布局规范

采用左侧固定侧边栏 + 顶部栏 + 主内容区的经典三栏布局：

```
┌─────────────────────────────────────────────────────┐
│ 顶部栏 (topbar)                                      │
│ [Logo]  [API 凭据切换]  [主题切换] [版本] [更新]      │
├────────┬────────────────────────────────────────────┤
│        │ 主内容区 (content)                          │
│ 侧边栏 │                                             │
│        │  ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│ 仪表盘 │  │  卡片   │ │  卡片   │ │  卡片   │       │
│ API凭据│  └─────────┘ └─────────┘ └─────────┘       │
│ 账号   │                                             │
│ 审计   │  ┌─────────────────────────────────┐       │
│ 设置   │  │         更多内容卡片              │       │
│ 安全   │  └─────────────────────────────────┘       │
│        │                                             │
└────────┴────────────────────────────────────────────┘
```

尺寸规范：
- 侧边栏宽度：240px（折叠时 64px）
- 顶部栏高度：56px
- 内容区内边距：24px
- 卡片间距：16px

## 3. 侧边栏规范

侧边栏包含以下导航项（按功能分组）：

**主要功能：**
- 🏠 仪表盘 — `/`
- 🔑 API 凭据 — `/credentials`
- 📱 账号会话 — `/accounts`
- 📋 审计日志 — `/audit`

**系统：**
- ⚙️ 系统设置 — `/settings`
- 🛡️ 安全说明 — `/security`

当前激活的导航项使用高亮背景色标识。

侧边栏底部显示：
- 当前登录用户（Phase 2 实现）
- 版本号

## 4. 顶部栏规范

顶部栏从左到右依次包含：
- Logo（Atria）
- API 凭据快速切换下拉框（Phase 3 实现）
- 主题切换按钮（浅色/深色/跟随系统）
- 当前版本号显示
- 检查更新按钮占位

## 5. 内容卡片规范

卡片是内容展示的基本单元：

```html
<div class="card">
    <div class="card-header">
        <h3 class="card-title">卡片标题</h3>
        <div class="card-actions">操作按钮</div>
    </div>
    <div class="card-body">
        卡片内容
    </div>
</div>
```

卡片样式：
- 背景色：`var(--card-bg)`
- 边框：`1px solid var(--border-color)`
- 圆角：`8px`
- 阴影：`var(--shadow-sm)`
- 内边距：`20px`

## 6. 表单规范

表单元素使用统一的样式：

- 输入框高度：40px
- 标签位于输入框上方
- 错误提示使用红色文字
- 禁用状态使用灰色背景
- 按钮分为：主要按钮（蓝色）、次要按钮（灰色）、危险按钮（红色）

## 7. 状态标签规范

使用标签（badge）显示状态信息：

- `enabled` — 绿色背景
- `disabled` — 灰色背景
- `active` — 绿色背景
- `banned` — 红色背景
- `restricted` — 橙色背景

## 8. 空状态规范

当列表无数据时显示友好的空状态提示：
- 图标或插图
- 说明文字
- 引导操作按钮（如有）

## 9. 错误提示规范

错误信息使用统一的样式：
- 错误页面使用居中卡片布局
- 显示错误代码和描述
- 提供返回首页的链接

## 10. 主题模式规范

支持三种主题模式：

### light（浅色）
- 默认主题
- 适合日间使用
- 白色背景 + 深色文字

### dark（深色）
- 适合夜间使用
- 深色背景 + 浅色文字
- 降低对比度以减少视觉疲劳

### system（跟随系统）
- 自动检测操作系统的主题偏好
- 使用 `prefers-color-scheme` 媒体查询
- 系统主题变化时自动切换

主题切换逻辑：
1. 用户选择的主题保存到 `localStorage`
2. 页面加载时读取偏好
3. `system` 模式监听 `matchMedia` 变化
4. 在 `<html>` 元素上设置 `data-theme` 属性

## 11. CSS 变量规范

所有颜色使用 CSS 变量管理，支持 light/dark 两套值：

```css
:root,
[data-theme="light"] {
    --bg-primary: #ffffff;
    --bg-secondary: #f5f5f5;
    --text-primary: #1a1a1a;
    --text-secondary: #666666;
    --border-color: #e0e0e0;
    --accent-color: #2563eb;
    /* ... 更多变量 */
}

[data-theme="dark"] {
    --bg-primary: #1a1a1a;
    --bg-secondary: #2d2d2d;
    --text-primary: #e0e0e0;
    --text-secondary: #999999;
    --border-color: #404040;
    --accent-color: #3b82f6;
    /* ... 更多变量 */
}
```

## 12. 不引入重前端框架的原因

Atria 不使用 React、Vue、Svelte、Tailwind、Bootstrap 等框架，原因如下：

1. **单二进制约束** — 所有资源必须通过 Go embed 嵌入，不需要 Node.js 构建链
2. **轻量部署** — 目标是自托管场景，不需要复杂的前端工程
3. **安全性** — 减少供应链攻击面
4. **可维护性** — 原生 HTML/CSS/JS 更容易理解和修改
5. **性能** — 无额外的运行时开销

允许使用的技术：
- Go html/template（服务端渲染）
- 原生 CSS（CSS 变量、Flexbox、Grid）
- 原生 JavaScript（ES6+）
- Go embed（资源嵌入）

## 13. 页面信息架构

| 页面 | 路由 | 说明 |
|------|------|------|
| 仪表盘 | `/` | 系统概览、统计卡片、快速操作 |
| API 凭据 | `/credentials` | 凭据列表、新增、编辑、删除 |
| 账号会话 | `/accounts` | 已登录账号列表、状态、操作 |
| 审计日志 | `/audit` | 操作日志列表、筛选 |
| 系统设置 | `/settings` | 系统配置、主题、更新 |
| 安全说明 | `/security` | 安全策略、风险说明 |
| 初始化 | `/init` | 首次管理员初始化（仅一次） |
| 登录 | `/login` | 管理员登录（Phase 2） |
