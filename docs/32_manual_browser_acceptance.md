# 人工浏览器验收清单

本文档列出自动化测试无法覆盖、需要真实浏览器验证的关键路径。

每次发布 main 前必须逐项验收。

## 前置条件

1. 启动 `bin/atria.exe serve`
2. 浏览器打开 `http://localhost:8080/app/`
3. 已登录管理员
4. 已接入至少一个 Telegram 账号
5. Runtime 状态为 live

---

## 1. Chat

| # | 验收项 | 操作 | 预期 |
|---|--------|------|------|
| 1.1 | 长历史会话默认显示最新消息 | 点击 Rabbit 等长历史会话 | 消息区显示最近消息，不显示历史顶部 |
| 1.2 | 上滑加载历史 | 在消息区向上滚动 | 自动加载更早消息，阅读位置保持 |
| 1.3 | 无滚动条时加载历史 | 消息不足一屏时向上滚轮 | 仍能触发加载更早消息 |
| 1.4 | 发送消息 | 在 composer 输入并发送 | 消息出现在底部，不重复 |
| 1.5 | 新消息实时同步 | 用手机给该账号发消息 | 不刷新页面，新消息自动出现 |
| 1.6 | 编辑消息同步 | 在 Telegram 编辑消息 | 对应消息自动更新 |
| 1.7 | 删除消息同步 | 在 Telegram 删除消息 | 对应消息自动消失 |
| 1.8 | sync 图标状态 | 观察消息头 sync 图标 | 连接中旋转，空闲时静止 |
| 1.9 | 切换账号不串消息 | 切换到另一个账号 | 显示新账号的会话和消息 |
| 1.10 | 新消息提示 | 上滑看历史时收到新消息 | 显示"有新消息"提示，点击回到底部 |

## 5. Accounts / Runtime

| # | 验收项 | 操作 | 预期 |
